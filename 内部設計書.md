## 内部設計書：シンプル・ワンタイム・アラームアプリ

本設計書は、外部設計書で定義されたシステム構成（UI層、データ管理層、システム制御層）に基づき、Pythonを用いた具体的なクラス構造、データフロー、およびアルゴリズムを定義する。

### 1. システムアーキテクチャとモジュール設計

システムは、責務に応じて以下の3つの主要なクラス/モジュールに分割し、明確に責務を分離する。

| モジュール名 | 責務 | 関連する層 |
| :--- | :--- | :--- |
| **`AlarmData` (Data Management)** | アラーム設定データの永続的な保存、読み出し、およびメモリ上の状態管理。 | データ管理層 |
| **`SystemController` (System Control)** | システム時刻の監視、タイマーの管理、およびアラーム鳴動のトリガー制御。 | システム制御層 |
| **`AlarmApp` (UI/Application)** | ユーザーインターフェースの表示、ユーザー入力の受付、および他のモジュール間の連携。 | UI層 |

### 2. データ構造の詳細設計

#### 2.1. アラーム設定データ (Data Management)

保存されるデータは、`alarm_settings.json` ファイルに格納される。

**JSON構造の確定:**
外部設計書に基づき、データは時刻と音の選択IDで構成される。

```python
# Pythonで扱われるデータ構造の定義
class AlarmSettings:
    """アラーム設定全体のデータ構造"""
    def __init__(self):
        # 時刻情報 (FR-01)
        self.hour: int = 0   # 0-23
        self.minute: int = 0 # 0-59
        
        # 音の選択 (FR-02)
        self.sound_id: int = 0 # 選択された音のID
```

#### 2.2. 音の管理 (Data Management)

システムが参照するアラーム音のリストとファイルパスを管理する。

*   **音のリスト:** アプリケーション起動時に、あらかじめ定義された音（例：`sound_list = ["sound_default.mp3", "sound_bell.wav", ...]`）のリストを保持する。
*   **音のIDとファイルパスの対応:** `sound_id` (整数) をキーとして、実際のファイルパスをマッピングする辞書を内部的に保持する。

### 3. データ管理層 (`AlarmData`) の詳細設計

このモジュールは、データ永続性（NFR-03）を担保する。

| メソッド名 | 処理内容 | 入力 | 出力 | 依存関係 |
| :--- | :--- | :--- | :--- | :--- |
| `load_settings()` | JSONファイルから設定を読み込み、メモリ上にインスタンスを構築する。 | ファイルパス | `AlarmSettings` オブジェクト | ファイルI/O |
| `save_settings(settings)` | メモリ上の設定をJSON形式に変換し、ファイルに書き込む。 | `AlarmSettings` オブジェクト | None | ファイルI/O |
| `get_settings()` | メモリ上の設定を返す。 | None | `AlarmSettings` オブジェクト | 内部メモリ |

**データフローの制御:**
1. **起動時:** `AlarmApp` が `AlarmData.load_settings()` を呼び出し、設定を初期化する。
2. **設定変更時:** `AlarmApp` が変更を適用した後、`AlarmData.save_settings()` を呼び出し、ディスクへの永続化を保証する。

### 4. システム制御層 (`SystemController`) の詳細設計

このモジュールは、時刻チェックとアラーム鳴動（FR-04, FR-03）を担当する。

#### 4.1. タイマー管理アルゴリズム

**動作:** システム時刻と設定時刻の差分を継続的に監視する。

1. **初期化:** `SystemController` は、`AlarmData` から設定時刻（H, M）とアラーム音IDをロードする。
2. **ループ開始:** アプリケーションが実行されている間、無限ループ（または定期的なスレッド）を開始する。
3. **時刻比較:**
    *   現在のシステム時刻 $T_{current}$ を取得する。
    *   設定時刻 $T_{alarm} = H \times 60 + M$ を計算する。
    *   差分 $\Delta t = T_{alarm} - T_{current}$ を計算する。
4. **トリガー判定:**
    *   もし $\Delta t \le 0$ で、かつアラームがまだ鳴動していない場合、**アラーム鳴動処理**を呼び出す。
    *   この処理後、アラーム状態を「鳴動済み」にマークし、次のトリガーを待つ。
5. **スリープ:** システムリソースの消費を抑えるため、比較間隔を一定に設定し、短いスリープ（例：1秒）を挟む。

#### 4.2. 音声再生インターフェース (TR-03)

外部システムとの連携は、OSのコマンドライン操作を通じて行う。

*   **関数:** `play_sound(file_path)`
    *   指定されたファイルパスのサウンドをOSのデフォルト再生機能を通じて再生する。

### 5. システム統合フロー

1. **初期化:** ユーザーインターフェースから設定値を受け取る。
2. **データロード:** 設定値を `AlarmData` に格納し、`play_sound` 関数を定義する。
3. **メインループ:** ユーザーからの入力を受け付けながら、時間経過を監視する。
4. **タイマー監視:** メインループ内で、現在の時刻と設定された時刻を比較する処理を定期的に実行する。
5. **イベント発生:** 時刻が一致した場合、`play_sound` 関数を呼び出して音を再生する。
6. **終了:** プログラムを終了する。

この設計により、要求された機能（設定、永続化、時間監視、外部操作）が、要求された制約（Python環境、ファイル操作）内で実現可能となる。
