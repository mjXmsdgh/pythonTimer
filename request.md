## 📝 要求定義書（たたき台）

### 1. プロジェクト概要

| 項目 | 内容 |
| :--- | :--- |
| **プロジェクト名** | シンプル・ワンタイム・アラームアプリ |
| **目的** | ユーザーが指定した時刻に、選択した音でアラートを鳴らす、シンプルなデスクトップ/組み込みアプリケーションを開発する。 |
| **開発言語** | Python |
| **環境** | Windows 11 または Raspberry Pi OS (Raspy) |

### 2. 機能要件 (Functional Requirements)

システムが**「何をすべきか」**を定義します。

| ID | 機能 | 詳細仕様 | 優先度 |
| :--- | :--- | :--- | :--- |
| **FR-01** | アラーム設定 | ユーザーは「時刻」（時・分）のみを設定できること。（日付は不要） | Must |
| **FR-02** | アラーム音の選択 | 設定時に、あらかじめ用意された複数の音の中からアラーム音を選択できること。 | Must |
| **FR-03** | タイマー操作 | ユーザーはアラーム設定後、タイマーを**停止（キャンセル）**できること。 | Must |
| **FR-04** | アラーム鳴動 | 設定時刻になった際、システムは**指定された時刻**に設定されたアラーム音を鳴らすこと。 | Must |
| **FR-05** | テスト機能 | アラーム音を鳴らす前に、設定した音で**短いテスト音**を鳴らすことができる機能を提供すること。 | Should |
| **FR-06** | データ保存 | 設定したアラーム情報（時刻、音の選択）は、**アプリケーションを終了しても保持される**こと。 | Must |

### 3. 非機能要件 (Non-Functional Requirements)

システムが**「どのように振る舞うべきか」**を定義します。

| ID | 要件 | 詳細仕様 | 優先度 |
| :--- | :--- | :--- | :--- |
| **NFR-01** | GUIのシンプルさ | ユーザーインターフェースは極めてシンプルで直感的であること。複雑な機能は排除する。 | Must |
| **NFR-02** | 動作環境 | Python環境で動作すること。GUIライブラリはシンプルで、Raspy/Win11での動作に最適化すること。 | Must |
| **NFR-03** | データ永続性 | 設定情報は、アプリを再起動しても保持されること。 | Should |
| **NFR-04** | 起動時の動作 | アプリケーション起動時、システムは**保存された設定を読み込み**、必要に応じてアラームのチェックを行うこと。 | Must |

### 4. システム・技術要件 (Technical Requirements)

システムが**「どのような制約で実現されるか」**を定義します。

| ID | 要件 | 詳細仕様 | 優先度 |
| :--- | :--- | :--- | :--- |
| **TR-01** | プラットフォーム | 開発初期段階では、クロスプラットフォーム対応（例：Tkinter）を優先し、Raspy/Win11での動作を確認する。 | Must |
| **TR-02** | GUIライブラリ | Python標準または広く使われているシンプルなライブラリ（例：Tkinter）を使用すること。 | Must |
| **TR-03** | 音声処理 | 音の再生には、Python標準機能またはOS標準のコマンド（例：`os.system` や `subprocess`）を利用し、外部ファイルの再生を実装すること。 | Must |
| **TR-04** | ファイル形式 | 設定データ（時刻、音の選択）は、**JSON**または**CSV**などのシンプルなテキストファイル形式で保存すること。 | Must |
