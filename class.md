このシステムは、**データ管理**、**時間処理**、**実行制御**の3つの主要な機能に分かれて設計されます。

---

## システムアーキテクチャ概要

システムは、以下の3つの主要なコンポーネントで構成されます。

1. **`AlarmData` (データ層):** アラームの設定値や状態を保持するデータ構造。
2. **`TimeManager` (時間処理層):** システムの時刻に基づき、アラームの有効性をチェックするロジック。
3. **`AlarmController` (制御層):** 入出力（I/O）を制御し、データと時間処理を連携させるメインの実行ロジック。

---

## 詳細なモジュール設計

### 1. AlarmData (データ構造)

このモジュールは、アラームに関連するすべての静的および動的なデータを保持します。

| 属性/メソッド | 説明 | 関連する要件 |
| :--- | :--- | :--- |
| **`__init__(self)`** | アラーム設定を初期化する。 | - |
| **`set_alarm(self, time_of_day, duration)`** | 特定の時刻と持続時間を設定する。 | 要件定義 |
| **`get_settings(self)`** | 現在設定されているアラーム情報を取得する。 | - |
| **`load_data(self, filepath)`** | 設定データをファイルから読み込む。 | 要件定義（永続化） |
| **`save_data(self, filepath)`** | 現在の設定データをファイルに保存する。 | 要件定義（永続化） |

**【役割】**
データの永続化（ファイル入出力）と、アラームのパラメータ（時刻、持続時間）の管理を担当します。

### 2. TimeManager (時間処理ロジック)

このモジュールは、現在のシステム時刻と設定されたアラーム時刻を比較し、アラームの状態を決定するロジックを提供します。

| 属性/メソッド | 説明 | 関連する要件 |
| :--- | :--- | :--- |
| **`check_status(self, current_time, alarm_data)`** | 現在時刻とアラーム設定を比較し、アラームがアクティブか判定する。 | - |
| **`calculate_remaining_time(self, start_time, end_time)`** | アラームの残り時間を計算する。 | - |

**【役割】**
時間に関するすべての計算と状態判定を担当します。システムが「今、アラームを発動すべきか」を判断するための核となります。

### 3. AlarmController (制御・実行層)

このモジュールは、ユーザーからの入力を受け取り、データ層と時間処理層を呼び出し、実際の出力（通知など）を実行するメインの制御ロジックです。

| 属性/メソッド | 説明 | 関連する要件 |
| :--- | :--- | :--- |
| **`__init__(self, data_manager, time_manager)`** | データ層と時間処理層のインスタンスを初期化する。 | - |
| **`run_system(self)`** | システムのメインループを実行する。 | - |
| **`process_input(self, input_data)`** | ユーザーからの入力を受け取り、`AlarmData`に保存する。 | 要件定義 |
| **`monitor_loop(self)`** | 継続的に現在の時刻を取得し、`TimeManager`を呼び出して状態を監視する。 | 要件定義 |
| **`trigger_alert(self, alert_details)`** | アラームが発動した際に、実際の通知（例：コンソール出力、外部通知）を実行する。 | - |

**【役割】**
システム全体の流れを統括し、他のモジュール間のデータの受け渡しと、最終的なアクションの実行を責任を持ちます。

---

## ワークフローフロー

システムは以下のフローで動作します。

1. **初期化:** `AlarmController`が`AlarmData`と`TimeManager`のインスタンスを初期化する。
2. **設定入力:** ユーザーが`AlarmController`に対し、アラーム設定（時刻、期間）を入力する。
3. **データ保存:** `AlarmController`は入力された設定を`AlarmData`に保存する。
4. **監視ループ開始:** `AlarmController`は`monitor_loop()`を実行し、システム時刻を継続的に取得する。
5. **状態チェック:** ループ内で、現在の時刻を`TimeManager`に渡し、`AlarmData`と照合してアラームの状態をチェックする。
6. **アクション実行:** 状態が「発動」と判定された場合、`AlarmController`は`trigger_alert()`を呼び出し、アラームを通知する。
7. **繰り返し:** ステップ4に戻り、システムは継続的に状態を監視し続ける。
